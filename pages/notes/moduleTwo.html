Networking Models

Identifiers

These are all used for web crawlers and to attach it to the css

NIT Cards

Used for direct connection from one computer to another
Essentially a Network Card

Hubs

Instead of wires from each nit card to each person, they all plug into a hub.
It blindly repeats to everyone what people are saying.

Ethernet

Standardized by IEEE

Encapsulation

TCP Sends segments
Generic term is Packet
Specific term is Frame at the ethernet layer
Only properly formed frames are allowed on the network

Collisions

2 people trying to talk at the same time
Destroys both frames
Unavoidable

CSMA/CD

Carrier sense multiple access collision detect
Listen before sending
Don't try to send until you're not recieving
Decreases the collisions

Loopback Circuts

Two waiting devices can decide to sent at the same time causing a collision

Collision recovery

Both send a jamming signal (not a valid frame)
Flood the whole network
Each sender generates its own random timer value
Both wait until their own timers expire before resending

Ethernet addresses

Between hub and loopbacks, all deices recieve al frames
Each device must determine which frames are meant for it
So each device must have a locally unique identity
Ethernet address (aka, MAC address)
Assigned by manufacturer and is completely unique
48 but calues - written as 12 hexadecimal digits, often grouped

Using the Ethernet Header

Contains both destination and source addresses
Separate fields
Header is both 802.3 & 802.2
Trailer is 802.3 only

FCS -> Frame Check Sqeuence
Generic -> Checksum

Switches

Allow multiple devices to send simultaneously
Examine destination of incoming frames and forward only to appropriate port
Result is that only sender and recipient are involved

Buffering

Buffers are temporary storage places
Used to store franes until they can be sent safely
Eliminates collisions
Takes less time than collision recovery

Flooding

Hubs send incoming frames out to all other ports
Switches behave the same way when the destination is not yet in their bridge table
Ensures destinatino device (as well as others) gets the frame

Broadcasting Frames

Most MAC addresses are unicast addresses
Refer to a specific destination device
ONe MAC address (all Fs) is the broadcast address
Frames sent to this address are flodded by the switch

Adaptable Switches

All varieties of ethernet use the same header so switches can handle multiple speeds
An autonegotiation standard allows the switch and any connected NICs to automatically determine the optimal speed for each connection
Full Duplex

Broadcast Domains

Collection of all devices that recieve broadcast frames sent by any one of them
Acts like a switch

Multiple separate LANs

Two separate LANs will require two separate switches
Each switch has its own address table
Each LAN is its own broadcast domain
Sometimes this behavior is desirable

Virtual LANs (VLANs)

Single switch supports multiple broadcast domains
Each broadcast domain is a separate VLAN
Uses a subset of the switches ports

Reasons to use VLANs

Limit each LAN to a smaller number of computers
Security
Creating TCP/IP subnets
Lower hardware expenses

Switch Stacks

Multiple Switches can be combined into a switch stack
Overcomes limit on number of available ports
No need to change switch logic

Switches in the stack add addresses on the other switches to their address table along with the port to the other switch

Cable between switches is called a trunk

VLAN Trunking

When VLANs involve a switch stack, frames sent over the trunk may be ambiguous
